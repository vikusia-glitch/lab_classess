Какое было количество вызовов конструкторов в обоих случаях?
В каком случае создаются временные объекты? Почему?
почему emplace_back работает эффективнее и в каких случаях?

1 vec.push_back(TrackedString("temp"))  2 конструктора: char* ctor для временного объекта, потом move ctor чтобы переместить его в вектор
vec.push_back(ts) 1 конструктор: copy ctor, потому что ts живёт дальше и мы не можем его забрать, только скопировать
vec.emplace_back("temp") 1 конструктор: char* ctor сразу внутри вектора
vec.emplace_back(TrackedString("temp")) 2 конструктора: char* ctor для временного объекта потом move ctor. то есть столько же сколько push_back

2 временный объект создаётся когда пишем TrackedString("temp") прямо внутри вызова это безымянный объект который живёт одну строку и он виден в логах потому что у него адрес в стеке, а не в векторе

3 push_back принимает уже готовый объект и кладёт его в вектор либо копируя либо перемещая, то есть объект сначала создаётся где-то, потом перекладывается
emplace_back принимает аргументы для конструктора и строит объект сразу на месте внутри вектора промежуточного объекта не
поэтому emplace_back("temp") эффективнее
