1. Объяснить разницу количества созданных объектов.
2. Объяснить, почему без списка инициализации создаётся лишний объект.
3. Объяснить роль временного объекта, зачем он появляется.
4. Объяснить (не)зависимость порядка объявления полей и их порядка сле-дования в списке инициализации, привести пример;
5. Как в списке инициализации по умолчанию инициализируются примитивы и поля-классы?


1 вариант а создает 2 объекта вариант б - 1
в первом варианте  логи [default ctor] id=1 и [string ctor] id=2 - сначала поле str автоматически создаётся конструктором по умолчанию
потом в теле конструктора создаётся временный объект TrackedString(s) через string ctor который присваивается полю str поэтому получается два объекта
во втором варианте где используется список инициализации поле str сразу создаётся через string ctor и лишнего объекта не возникает

2 все поля класса обязательно инициализируются до выполнения тела конструктора и если поле не указано в списке инициализации, компилятор автоматически вызывает конструктор по умолчанию
в варианте апервый объект был создан впустую потому что его значение сразу перезаписывается

3 str = TrackedString(s) значает: сначала создали безымянный временный объект типа TrackedString из строки s а затем присвоили его в уже существующее поле str через operator=
временный объект это анонимный объект без имени, который живёт ровно одну строку: создаётся для передачи значения и сразу уничтожается после присваивания он появляется потому что в теле конструктора str уже существует и единственный способ изменить его значение это присваивание а не конструирование
в варианте Б временного объекта нет потому что список инициализации конструирует str сразу с нужным значением присваивания не происходит

4 поля инициализируются в порядке их объявления в классе, а не в порядке, в котором они написаны в списке инициализации
наприме "class х {
int a;
int b;
public:
х(int x) : b(x), a(b) {}
};" - несмотря на то что b написано раньше сначала инициализируется a
в этот момент b ещё не имеет значени поэтому a получает мусор

5 если поле-класс не указано в списке инициализации вызывается конструктор по умолчанию
если примитив не упомянуть в списке инициализации он не инициализируется и будет содержать мусор (неопределённое значение)
